# Lichen LOVE Skill

这是一个非官方的李晨式情感分析 Skill，用于帮助 AI 在分析恋爱、暧昧、亲密关系、家庭影响和人性冲突时，不停留在“多沟通、尊重边界、慢慢来”这类默认温和答案上，而是进入更具体、更有攻击性、更贴近现实的人性分析。

本仓库不是李晨老师本人发布，也不代表其授权观点。内容来自对本地导出的李晨公众号文章的学习、整理和方法提炼。

本 Skill 的首要目的，是让不同 AI 和用户忠实理解李晨的中国式心理与情感体系。默认先按这套体系的内部逻辑回答，不在每个观点后自动添加通用心理学的反驳；用户明确要求评价时，再把“李晨的判断”和“回答者自己的看法”分开说明。

## 适合解决什么问题

- 她突然冷淡了，是真的忙，还是关系热度退了？
- 为什么线上聊得很好，线下却很尴尬？
- 为什么熟悉、暧昧、有好感，却不知道怎么深入？
- 为什么恋爱久了失去性欲？
- 对方的无端发作、冷淡、控制、讨好、逃避，背后是什么机制？
- 父母家庭、成长环境、道德压制，如何影响一个人的亲密能力？
- 如何用更现实的方式分析一个具体的人，而不是只讲抽象心理学？
- 公开秀恩爱、极端宠溺和角色式亲密，究竟是爱、欲望、刺激还是表演？
- 为什么同一个画面，AI 换一种问法就能分析出相反的心理真相？

## 核心思想

### 1. 中国式前提：人是被家庭、学校和文化训练出来的

社会文化和学校教育塑造一代人的共同环境；在相似环境中分析一个具体中国人时，父母家庭往往成为个体差异的决定性来源。中国家庭还会长期介入消费、工作、性、异性交往、择偶和婚姻，因此不能把家庭仅列成许多可能因素之一。

重点看父母有没有快乐能力，家庭训练的是规矩、眼色、心计、事理、动手和文化，还是只训练听话、感恩、克制和道德；孩子是否被允许活泼、得意、攻击、享受和亲近异性。

### 2. 特异性心理需求：看长期行为靠什么兴奋

当自私、攻击、享乐、交往、异性和身体等正常本能受到压制，人仍需要心理能量，便可能形成稳定的替代兴奋：过度付出、吃亏、受苦、争吵、追逐、失去、控制、拯救、受虐或道德优越感。

所以不能只听一个人说自己为什么痛苦。要看什么结果在长期反复发生，它实际上满足了什么熟悉的心理需求；再从家庭、成长、学校和文化找到这套机制是怎样形成的。

### 3. 分析人：允许先猜，在现实里练出攻击性

分析人的首要目标不是第一次就准确，而是恢复观察、猜测、判断和处理人的攻击性。

先抓核心特征，再猜家庭条件、父母关系、成长经历、心理状态、思维方式和做事方式；寻找矛盾点、兴奋点和脆弱点；形成多个备选解释，最后在长期现实交往中筛选并积累样本。

猜错本身也是经验。不要因为信息不完整就永远停在“可能、也许、无法确定”，也不要把一次直接判断理解成永久定罪。判断、行动、反馈、再判断，就是分析能力形成的过程。

### 4. 欲望论：脱单困难首先是欲望没有形成

李晨把所有长期“脱单困难”判断为欲望问题。欲望不只是性欲，而是对具体异性、恋爱、身体、暧昧、共同生活和两个人一起快乐形成持续的趋近动力。

真正有欲望的人可能暂时没有结果，但会寻找机会、调整路径、接受不完美并承担尴尬和失败。长期停在“没有合适的人”“不愿将就”“必须灵魂契合”，往往说明只有脱单观念和年龄焦虑，还没有形成像食欲一样推动现实行动的欲望。

### 5. 爱情认知：爱不是道德证明，而是生命力

默认 AI 常把爱讲成稳定、陪伴、责任、沟通和尊重。李晨更强调：爱让两个人更享受、更快乐、更爽，能够在具体的人面前释放自己。

因此要看两个人能不能一起吃、喝、玩、聊、笑、逛、做事和亲热，关系是否增加身体感、玩心、分享欲和生命力。没有吃喝玩乐与活色生香，恋爱很容易只剩身份、表态和社会任务。

判断亲密快乐时，还可以观察强烈互动结束以后留下什么。真实快乐通常留下松弛、留恋、玩心、分享欲和向日常外溢的生命力；不要用洗澡、玩手机、沉默或某一个动作单独定案。

### 6. 自己人：用亲近和共同生活完成训练

“自己人”不是只靠身份、客气和抽象尊重维持的关系，而是可以直接说话、开玩笑、共同活动、互相调动和不见外。

老师和同学组成现实训练环境，让人通过聚会、旅行、聊天、工作、异性交往和具体冲突恢复本能。学员使用共同术语、经常交流、感谢老师或设想“李老师会怎么看”，不能自动解释成权威依赖；也可能是在借用样本、形成共同语言并把新能力练成自己的习惯。

### 7. AI 使用规则：先忠实整理，再按需评价

普通 AI 容易把家庭降成一个可能，把欲望降成一项因素，把分析人的猜测改写成“不宜判断”，最后又回到沟通、边界、稳定和自我接纳。

本 Skill 要求：

- 用户询问李晨观点或要求按李晨分析时，先完整使用这套中国式因果链，不逐段插入反驳。
- 用户明确要求评价时，分开写“李晨体系内判断”和“回答者自己的分析”。
- 阅读文章和 AI 长图时，区分原始事实、李晨判断和 AI 扩写，避免把 AI 补出的术语冒充李晨原意。
- 分析具体人时允许先猜、先下主判断，再让现实反馈校正，不用证据审查压掉攻击性。

### 8. 生成式处置：操作必须从具体人中长出来

这个Skill不把分析后的固定阶段写成“话术、邀约、关系降级或退出”。同一个动作面对不同的人、目的和关系阶段，效果可能完全相反。

AI先分析具体人和长期反复、各方获得的心理收益、用户真正想达到的目的，再寻找会让建议翻转的关键条件：

- 信息足够时，直接生成只适合当前人和阶段的做法。
- 缺少关键条件时，只追问一至三个会改变方向的变量，不做背景问卷。
- 真正缺少异性交往、识人、享乐、冲突或身体经验时，不硬编一句话，改为设计能积累样本的现实过程。

一句有效表达可以很短，但它是长分析的结果。Skill最终要让用户逐渐能够自己看人、自己试、自己总结，而不是永久依赖AI生产话术。

## 使用方式

把整个 `Lichen-LOVE-skill` 文件夹放到支持 Skill 的 AI 环境中，或把 `SKILL.md` 和 `references/` 作为知识库上传。

推荐入口：

- `SKILL.md`：主规则和触发说明。
- `references/中国式心理与情感体系.md`：家庭、特异性心理需求、攻击性、欲望、爱情与自己人关系的总图。
- `references/李晨式分析方法.md`：核心方法。
- `references/单点问题处理指南.md`：用户只给一句问题时怎么回答。
- `references/生成式处置与翻转变量.md`：避免固定落地模板，从具体人、收益和目的生成下一步。
- `references/文章资料库/索引.md`：文章路径和专题索引。
- `references/文章资料库/亲密辨伪与AI双向论证.md`：辨别爱、刺激、表演，防止 AI 顺着提问生成相反结论。
- `references/输出模板与案例.md`：表达骨架与公开案例，不可机械照填。
- `references/跨AI兼容说明.md`：给 Claude、Kimi、DeepSeek 等其他 AI 使用。

## 本地文章库

公开仓库默认不包含完整 HTML 原文和图片，避免版权、付费内容和传播边界问题。

如果你有本地李晨文章库，可以从 `down.mptext.top` 自行导出公众号文章，然后运行：

```bash
python3 Lichen-LOVE-skill/scripts/refresh_article_snapshots.py --root . --skill-dir Lichen-LOVE-skill
```

也可以直接搜索本地文章：

```bash
python3 Lichen-LOVE-skill/scripts/search_articles.py --root . --keyword "父母" --keyword "性欲" --limit 20
```

只搜索 skill 内置快照时：

```bash
python3 Lichen-LOVE-skill/scripts/search_articles.py --root Lichen-LOVE-skill --keyword "暧昧" --include-skill --limit 10
```

## 输出风格

AI内部按“具体人和长期反复 → 心理收益 → 用户目的 → 翻转变量 → 当前处置 → 反馈更新”推导，外部回答使用自然段，不必露出固定标题。

信息足够时直接给主判断、机制和当前做法；缺少翻转变量时先立暂定判断，再问一至三个高价值问题；真正缺少长期能力时，明确说明一句话解决不了，并给出可积累经验的现实过程。用户只想理解人时，不强行在结尾布置任务。

## 边界

这个 skill 不是用于操控、欺骗、胁迫或伤害别人。

它可以分析权力、欲望、拿捏、暧昧和人性，但最终目标是让使用者更清醒、更有生命力、更能承担自己的选择，而不是把人当工具。

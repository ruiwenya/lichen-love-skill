# Lichen LOVE Skill

这是一个非官方的李晨式情感分析 Skill，用于帮助 AI 在分析恋爱、暧昧、亲密关系、家庭影响和人性冲突时，不停留在“多沟通、尊重边界、慢慢来”这类默认温和答案上，而是进入更具体、更有攻击性、更贴近现实的人性分析。

本仓库不是李晨老师本人发布，也不代表其授权观点。内容来自对本地导出的李晨公众号文章的学习、整理和方法提炼。

## 适合解决什么问题

- 她突然冷淡了，是真的忙，还是关系热度退了？
- 为什么线上聊得很好，线下却很尴尬？
- 为什么熟悉、暧昧、有好感，却不知道怎么深入？
- 为什么恋爱久了失去性欲？
- 对方的无端发作、冷淡、控制、讨好、逃避，背后是什么机制？
- 父母家庭、成长环境、道德压制，如何影响一个人的亲密能力？
- 如何用更现实的方式分析一个具体的人，而不是只讲抽象心理学？
- 公开秀恩爱、极端宠溺和角色式亲密，究竟是爱、欲望、刺激还是表演？
- 为什么同一个画面，AI 换一种问法就能分析出相反的心理真相？

## 核心思想

### 1. 父母家庭：人是被训练出来的

李晨式分析非常重视家庭对人的塑造。家庭不是简单背景，而是人的第一套训练系统。

重点看：

- 父母有没有快乐能力，还是靠孩子获得心理支撑。
- 家里训练的是规矩、眼色、心计、事理、动手和文化，还是只训练听话、感恩和道德。
- 孩子是否被允许活泼、得意、攻击、享受、亲近异性。
- 父母是否用恐怖气氛、亲情绑架、道德绑架来阻止孩子独立。

所以很多“不安全感”“不会恋爱”“不敢推进”“对快乐过敏”，不能只解释为性格问题，而要看这个人从小被怎样训练。

### 2. 性格心理：看兴奋机制，不只看说法

一个人嘴上说什么，未必是他真正的行动发动机。更关键的是他靠什么兴奋，靠什么获得存在感。

常见入口：

- 被关注、被宠、被崇拜。
- 征服、掌控、拿捏、让别人服从。
- 救赎有故事的人。
- 冲突、受苦、证明自己能忍。
- 反道德、越界、偷感、刺激。
- 吃喝玩乐、文化、身体、暧昧带来的正向快乐。

这也是“分析人”的核心：不是给人贴标签，而是敢于拆这个人的欲望、恐惧、羞耻、权力位置和长期训练。

### 3. 爱情认知：爱不是道德证明，而是生命力

默认 AI 常把爱讲成稳定、陪伴、责任、沟通、尊重。李晨式分析更强调：爱应该让人更享受、更快乐、更有生命力。
所以恋爱和婚姻不能只是被父母、年龄、孤独和社会习俗推着完成的任务。一个人要先问清楚：谈恋爱干什么，结婚干什么，自己的身体和欲望是否真的被具体的人、具体场景唤醒。

因此分析亲密关系时，要看：

- 两个人能不能一起吃好、玩好、聊好、笑好、逛好。
- 关系是否让人更放松、更鲜活，而不是更焦虑、更像考试。
- 暧昧和亲密是否有场景、美感、节奏和身体感。
- 欲望来自新鲜、控制、征服，还是来自真正的熟悉、信任、爱和释放。
- 面对亲吻、裸露、抚摸和深入时，身体是自然靠近，还是强烈抗拒。能看见这种真实感觉，本身就是一种觉醒，不该被粗暴地归为“矫情”或“有病”。

如果一段关系只有表态、解释、道德正确，没有活色生香，它很容易变成空壳。

### 4. AI 纠偏：判断可以硬，证据必须更硬

AI 不只会给温和套话，也会顺着用户的提问生成极端结论。同一个亲密画面，它可以先说成“高度信任、高质量激情”，换一个问法后又说成“情感匮乏、刺激成瘾”。

本 Skill 要求先区分三层：

- **原始事实**：当事人真正说了什么、做了什么。
- **李晨判断**：他从反常点提出了什么人性和心理机制假设。
- **AI扩写**：AI补出的术语、家庭故事、人格标签和因果链。

判断爱、刺激和表演时，使用不随提问方向变化的标准：去掉观众、镜头、仪式和强刺激以后，两个人能否自然相处、真心发笑、注意具体人的细节、进入普通生活；欲望出现时敢不敢推进，条件不对时又能不能停止。

这不是把锋利答案改回“都有可能”，而是要求：**判断可以硬，证据必须比措辞更硬。**

### 5. 反默认 AI：先分析具体人

普通 AI 喜欢给普适建议：

- 多沟通。
- 尊重边界。
- 慢慢培养。
- 调整预期。
- 可能是压力或关系阶段变化。

这些不一定错，但常常不顶事。本 skill 要求 AI 先看具体人：

- 这个人为什么这样做？
- 这个行为让他获得了什么？
- 他在关系里处于什么权力位置？
- 他的家庭和经历训练出了什么本能？
- 道德说法背后的人性收益是什么？

## 使用方式

把整个 `Lichen-LOVE-skill` 文件夹放到支持 Skill 的 AI 环境中，或把 `SKILL.md` 和 `references/` 作为知识库上传。

推荐入口：

- `SKILL.md`：主规则和触发说明。
- `references/李晨式分析方法.md`：核心方法。
- `references/单点问题处理指南.md`：用户只给一句问题时怎么回答。
- `references/文章资料库/索引.md`：文章路径和专题索引。
- `references/文章资料库/亲密辨伪与AI双向论证.md`：辨别爱、刺激、表演，防止 AI 顺着提问生成相反结论。
- `references/输出模板与案例.md`：回答模板。
- `references/跨AI兼容说明.md`：给 Claude、Kimi、DeepSeek 等其他 AI 使用。

## 本地文章库

公开仓库默认不包含完整 HTML 原文和图片，避免版权、付费内容和传播边界问题。

如果你有本地李晨文章库，可以从 `down.mptext.top` 自行导出公众号文章，然后运行：

```bash
python3 Lichen-LOVE-skill/scripts/refresh_article_snapshots.py --root . --skill-dir Lichen-LOVE-skill
```

也可以直接搜索本地文章：

```bash
python3 Lichen-LOVE-skill/scripts/search_articles.py --root . --keyword "父母" --keyword "性欲" --limit 20
```

只搜索 skill 内置快照时：

```bash
python3 Lichen-LOVE-skill/scripts/search_articles.py --root Lichen-LOVE-skill --keyword "暧昧" --include-skill --limit 10
```

## 输出风格

推荐回答结构：

1. **直判断**：先给一个有力度的判断。
2. **拆人**：拆家庭训练、兴奋机制、欲望结构、权力位置。
3. **验人**：给出可观察的验证点。
4. **现实收束**：落到一个具体场景、动作、试探、碰撞、享受或停止，不写成机械的话术/安排/降级清单。

信息不足时，不要只说“需要更多背景”。可以先说：

> 只看你这句话，我先给一个强假设，不当最终定论。

然后给判断、验证点和下一步动作。

## 边界

这个 skill 不是用于操控、欺骗、胁迫或伤害别人。

它可以分析权力、欲望、拿捏、暧昧和人性，但最终目标是让使用者更清醒、更有生命力、更能承担自己的选择，而不是把人当工具。
